# FX Portal Mobile Design Spec

作成日: 2026-06-18  
目的: AUXEN FX Portal をスマホ閲覧前提でも読みやすく、使いやすい情報サイトにする。

## 背景

PC版のheroデザインは整ってきたが、スマホではPC向けレイアウトを折りたたんだだけになっている。
そのためファーストビューの大半をロゴ、ナビ、heroが占め、ユーザーが本当に見たい「今日の日報」「重要指標」「通貨ランキング」へ到達しにくい。

FX情報サイトはスマホ流入が多い前提で考える。
通勤中、休憩中、指標前、チャート確認中に短時間で見る用途が中心になるため、スマホ版は装飾よりも情報導線を優先する。

## デザイン方針

PC版:
- ブランド感、研究所感、高級感を優先する
- 横長heroとサイドバーでAUXENらしさを出す
- ダッシュボードとして複数情報を俯瞰できる

スマホ版:
- 速報性、タップしやすさ、読みやすさを優先する
- ファーストビューで「今日読むべき情報」に到達できる
- PC用の大きなheroをそのまま使わない
- 羊飼いのFXさんのように、メニュー入口と主要情報を密度高く並べる

## スマホ版のファーストビュー構成

推奨順:

1. Compact Header
   - 左: AUXENロゴ + `FX Research Lab`
   - 右: メニューアイコン
   - 高さは80px程度まで
   - PC版サイドバーはスマホでは表示しない

2. Mobile Hero / Latest CTA
   - 高さは180〜220px程度
   - 見出し: `AIと統計で相場を研究する`
   - サブコピーは1行または2行まで
   - `LAST UPDATE` はカードではなく、小さな1行メタ情報にする案を優先
     - 例: `更新: 2026-06-18 / 毎朝7時`
   - 背景波形は下部に薄く残す
   - 大きな右側date-cardはスマホでは非表示または小型化

3. Quick Menu Grid
   - 2列グリッド
   - タップ領域を十分大きくする
   - 最低限の入口:
     - 最新日報
     - 今日の重要指標
     - 通貨ランキング
     - 政策金利
     - 日報アーカイブ
     - お問い合わせ

4. Latest Report Card
   - 日付、日報タイトル、一言まとめ、主要リスク、CTAを表示
   - スマホでは4枚summaryカードを最初から全部並べない
   - 重要な1〜2項目に絞る

5. Sticky Bottom Nav
   - 下部固定ナビを検討
   - 候補:
     - Home
     - 日報
     - 指標
     - メニュー

## スマホで非推奨の現状

- サイドバーを上部にそのまま縦表示する
- 横スクロールナビに主要導線を詰め込む
- hero内に大きなdate-cardを残す
- PC版と同じsummary-gridをそのまま縦積みにする
- ファーストビューで最新日報の本文入口が見えない

## 実装方針

同じURL、同じHTMLを使い、CSSと一部スマホ専用HTMLで出し分ける。
別URLや別サイトにはしない。

想定ブレイクポイント:
- PC: 961px以上
- Tablet: 561px〜960px
- Mobile: 560px以下

追加する主な構造案:

```html
<header class="mobile-header">
  <a class="mobile-brand" href="index.html">...</a>
  <button class="mobile-menu-button">...</button>
</header>

<section class="mobile-hero">
  <p class="eyebrow">AUXEN FX PORTAL</p>
  <h1>AIと統計で相場を研究する</h1>
  <p>日報・重要指標・政策金利を毎朝更新。</p>
  <p class="mobile-update">更新: 2026-06-18 / 毎朝7時</p>
</section>

<nav class="mobile-quick-grid">
  <a href="#latest-report">最新日報</a>
  <a href="#today-events">重要指標</a>
  <a href="#ranking">通貨ランキング</a>
  <a href="#rates">政策金利</a>
  <a href="#reports">アーカイブ</a>
  <a href="contact.html">お問い合わせ</a>
</nav>
```

CSS方針:
- `.sidebar` はスマホで非表示または折りたたみメニュー化
- `.hero` はPC専用として扱い、スマホでは `.mobile-hero` を表示
- `.summary-grid` はスマホでは2列または重要項目だけにする
- `.report-feature` はスマホで余白を減らし、CTAを目立たせる
- 文字サイズは viewport width 連動ではなく固定/段階指定にする

## 情報設計

スマホで最優先する情報:

1. 今日の日報を読む
2. 今日の重要指標を確認する
3. 今日の注目通貨/通貨ランキングを見る
4. 政策金利を確認する
5. 過去日報へ移動する

スマホで優先度を下げる情報:

- ブランド説明
- Coming Soon系の大きなプレースホルダー
- 長い政策金利テーブル
- PC向けの装飾的hero

## 受け入れ条件

- 390px幅のスマホ表示で、ファーストビュー内に以下が見える
  - AUXENロゴ
  - コンパクトhero
  - 最新日報またはQuick Menuの入口
- `LAST UPDATE` が大きなカードとして画面を圧迫しない
- 横スクロールしない
- 主要ナビが途中で見切れない
- 最新日報CTAを1タップで押せる
- PC版heroの見た目は大きく崩さない
- 自動生成される日報ページにもスマホ導線の考え方を反映する

## 実装ステップ案

1. `index.html` にスマホ専用header/hero/quick-gridを追加
2. `style.css` にスマホ専用CSSを追加
3. PC版表示への影響を確認
4. `reports/*.html` の日報ページにもスマホhero/ナビ方針を反映
5. `trigger_prompt.txt` にスマホ構造の生成ルールを追加
6. ローカルChromeで 390px / 430px / 768px / 1440px を確認
7. ユーザープレビューOK後にcommit/push

## レビュー観点

別AIには以下を重点的に確認してもらう。

- スマホファーストの情報設計として妥当か
- FX情報サイトとして、最初に出すべき情報の優先順位が正しいか
- 羊飼いのFXさんのような「タップしやすい入口」になっているか
- PC版のブランド感とスマホ版の実用性の両立ができそうか
- 実装時に自動日報生成が壊れないか
